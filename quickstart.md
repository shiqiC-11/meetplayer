# MeetPlayer 快速上手指南

欢迎开始 MeetPlayer 项目开发！本指南将帮助你在 5 分钟内运行起项目。

## 🚀 快速开始（5分钟）

### Step 1: 打开微信开发者工具（1分钟）

1. 启动微信开发者工具
2. 扫码登录
3. 点击 **导入项目**

### Step 2: 导入项目（30秒）

- **项目目录：** 选择 `meetplayer` 文件夹
- **AppID：** 
  - 如果有小程序账号：输入你的 AppID
  - 如果没有：选择 **测试号**（有部分功能限制）
- **项目名称：** MeetPlayer
- 点击 **导入**

### Step 3: 构建 npm（1分钟）

在微信开发者工具中：

1. 点击菜单栏：**工具** → **构建 npm**
2. 等待构建完成（会生成 `miniprogram_npm` 目录）

> ✅ 看到 "构建 npm 完成" 提示即可

### Step 4: 开通云开发（2分钟）

1. 点击顶部的 **云开发** 按钮
2. 根据提示开通云开发
3. 创建云开发环境：
   - 环境名称：`dev`
   - 点击 **创建**
4. 等待环境创建完成（约1分钟）

### Step 5: 配置环境 ID（30秒）

1. 在云开发控制台，点击 **设置** 标签
2. 复制 **环境 ID**（格式：`cloud1-xxx`）
3. 编辑 `app.js` 文件第 9 行：

```javascript
wx.cloud.init({
  env: 'your-env-id-here', // 👈 粘贴你的环境 ID
  traceUser: true
});
```

4. 保存文件（Ctrl+S / Cmd+S）

### Step 6: 运行项目（10秒）

点击微信开发者工具顶部的 **编译** 按钮

🎉 **完成！** 你应该能看到小程序运行起来了。

## ⚠️ 常见问题

### Q: 点击"编译"后报错？

**检查清单：**
- [ ] 是否已经构建 npm？
- [ ] `app.js` 中的环境 ID 是否正确？
- [ ] 网络是否正常？

### Q: 地图不显示？

**原因：** 需要授权位置权限

**解决：** 在模拟器设置中授权位置权限，或使用真机预览

### Q: TabBar 图标不显示？

**原因：** 缺少图标文件

**临时解决：** 编辑 `app.json`，注释掉 TabBar 的图标配置：

```json
"tabBar": {
  "list": [
    {
      "pagePath": "pages/index/index",
      "text": "附近"
      // "iconPath": "assets/icons/nearby.png",
      // "selectedIconPath": "assets/icons/nearby-active.png"
    }
  ]
}
```

### Q: 点击功能后显示"开发中"？

**原因：** 这是正常的，Phase 1 只完成了基础框架

**下一步：** 继续开发 Phase 2 的核心功能

## 🛠️ 接下来做什么？

### 选项 A: 继续开发核心功能

参考 [SETUP.md](./SETUP.md) 和 [PROJECT_SUMMARY.md](./PROJECT_SUMMARY.md)

**建议顺序：**
1. 创建数据库集合
2. 开发云函数
3. 完善首页数据展示
4. 实现发布功能

### 选项 B: 添加测试数据

在云开发控制台手动添加一些测试数据，让页面"活"起来：

1. 创建 `venues` 集合
2. 添加几个测试球场
3. 创建 `slots` 集合
4. 添加几个测试约球局

### 选项 C: 自定义样式

修改 `app.wxss` 中的颜色变量：

```css
page {
  --primary-color: #52C41A;  /* 主色 */
  --secondary-color: #F5F5F5; /* 背景色 */
}
```
